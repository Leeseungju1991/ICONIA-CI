# 국외이전 점검표 (Cross-Border Transfer Checklist)

> 개인정보 보호법 제28조의8 의 국외이전 정당화 사유 점검표.

## 1. 현재 국외이전 항목

| Item | 이전 국가 | 이전 받는 자 | 이전 데이터 | 이전 목적 |
|---|---|---|---|---|
| AI-01 | 미국 | Google LLC | 사용자 입력 메시지 + 페르소나 컨텍스트 (PII redact 후) | Gemini API 응답 생성 |

## 2. 정당화 사유 (개인정보 보호법 제28조의8)

| 사유 | 적용 여부 (AI-01) | 근거 |
|---|---|---|
| 1) 정보주체의 동의 | ✓ | CN-04 + CN-05 동의 — `users.additional_consents.cross_border` |
| 2) 법령에 특별한 규정 | n/a | — |
| 3) 협정 (한미 FTA 등) | n/a | — |
| 4) 정보주체와 계약 이행 | n/a | — |
| 5) 정보주체 또는 제3자 생명·신체·재산상 이익 | n/a | — |
| 6) 보호위원회 인증 (해외 사업자) | (검증 필요 — HD-18) | Google 의 보호위원회 인증 여부 |
| 7) 동등 수준 보호 인정 국가 (보호위원회 고시) | (검증 필요 — HD-18) | 미국이 동등 수준 인정 국가인지 확인 |

## 3. 사용자 고지 사항 (보호법 제28조의8 제2항)

다음 사항을 사용자에게 명확히 고지한다:

- ✓ 이전 받는 자: Google LLC
- ✓ 이전 받는 자의 위치: 미국
- ✓ 이전 받는 자의 개인정보 처리 정책: https://policies.google.com/privacy
- ✓ 이전 받는 자의 보호조치: Google 의 자체 보호 정책 + 회사가 PII redaction 적용
- ✓ 이전 받는 자의 개인정보 책임자 연락처: Google 정책 페이지 참조
- ✓ 이전 받는 자의 거부 가능성: AI 기능 미사용 가능 (선택 거부)

## 4. 안전성 확보 조치

| 조치 | 내용 |
|---|---|
| 전송 보안 | TLS 1.2+ HTTPS |
| 데이터 마스킹 | PII redaction (이메일, 전화, 주민번호 등) 후 전송 |
| 최소 정보 원칙 | 페르소나 응답에 필요한 최소 컨텍스트만 전송 |
| 보관 기간 통제 | Google 정책에 따르되, 회사 측 캐시는 30일 |
| 사고 대응 | Google 측 사고 발생 시 사용자에게 통지 — `docs/operations/personal-data-breach-runbook.md` |

## 5. LEGAL_REVIEW_REQUIRED (HD-18)

- Google 의 한국 사용자 데이터 보호 정책 적정성
- 동등 수준 인정 국가 여부 (보호위원회 고시 확인)
- 보호위원회 인증 사업자 여부

## 6. 운영 점검

- 분기별 1회 — 본 점검표 갱신
- 위탁 사업자 (AWS, Google) 정책 변경 시 즉시 갱신
