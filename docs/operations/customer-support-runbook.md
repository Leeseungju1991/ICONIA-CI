# Customer Support Runbook

## 1. 접수 채널

- 앱 내: 설정 > 문의하기 (이메일 링크)
- 이메일: support_email (HD-01)
- 회사 홈페이지 문의 폼 (해당 시)

## 2. 주요 응대 카테고리

| Category | Typical Action | SLA |
|---|---|---|
| 계정 / 로그인 문제 | ADMIN 에서 사용자 조회 + 강제 로그아웃·MFA reset | 영업일 1일 |
| 디바이스 페어링 실패 | 디바이스 진단 (last_seen_at, 펌웨어 버전) + 운영자 가이드 | 영업일 1일 |
| AI 응답 문제 (부적절한 응답) | 신고 접수 → 운영팀 검토 → 페르소나 조정 또는 차단 | 영업일 2일 |
| 결제 / 환불 문의 | "앱 내 결제 미제공" 안내 + 외부 채널 안내 | 영업일 1일 |
| 개인정보 문의 | privacy_officer_email 로 전달 + 본 runbook 의 권리 요청 처리 | 보호법 SLA |
| 피드 신고 / 모더레이션 | ADMIN `community` 페이지에서 처리 | 영업일 1일 |
| 펌웨어 업데이트 실패 | OTA 로그 확인 + 수동 재시도 가이드 | 영업일 1일 |
| 기타 버그 | 재현 절차 수집 → 개발팀 ticket | 영업일 2일 |

## 3. 표준 응대 예시

### 결제 / 환불 문의 (display-only 정책)

> 안녕하세요, ICONIA 고객지원입니다.
>
> 현재 ICONIA 앱은 상품 안내 목적으로만 운영되며, 앱 내에서 결제·주문·환불 기능을 제공하지 않습니다.
>
> 실제 구매는 회사 별도 채널을 통해 이루어집니다. (HD-04 결정 후 외부 채널 안내)

### AI 응답 부적절 신고

> 안녕하세요, ICONIA 고객지원입니다.
>
> 신고하신 응답을 검토하겠습니다. AI 페르소나 응답은 인공지능이 생성하는 것으로 의도하지 않은 결과가 나올 수 있으며, 회사는 사용자 피드백을 바탕으로 지속적으로 개선하고 있습니다.
>
> 신고 결과 처리 후 별도 통지드리겠습니다.

## 4. 운영자 처리 흐름

1. 이메일/문의 접수 → CS 운영자 ADMIN 로그인
2. `dashboard/cs` 페이지에서 사용자/디바이스/오류 통합 조회
3. 처리 방안 결정 (위 표 참조)
4. 처리 완료 후 사용자에게 이메일 응답
5. 처리 내역 `audit_logs` 자동 기록

## 5. 에스컬레이션

- P0 (즉시 처리): 보안 사고, 개인정보 침해, 다수 사용자 영향
- P1 (24시간 내): 결제·환불 (해당 시), 펌웨어 위험, 법적 분쟁
- P2 (영업일 1일): 일반 문의
- P3 (영업일 3일): 개선 요청, 제안

## 6. 사후 분석

- 월 1회 CS 통계 보고서 작성 (DPO 에게 개인정보 관련 항목 별도 보고)
- 자주 발생하는 문의 → FAQ 화면 / 도움말 보강
