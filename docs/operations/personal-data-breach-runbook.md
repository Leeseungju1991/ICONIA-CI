# 개인정보 침해사고 대응 Runbook

> 개인정보 보호법 제34조 (개인정보 유출 통지 등) + 시행령 제40조 기준.

## 1. 정의

다음 중 하나에 해당하면 침해사고로 본다:

- 외부 공격으로 개인정보 DB 접근 가능성 발견
- 운영자 계정 탈취 또는 의심
- 백업 데이터의 외부 노출
- 위탁 처리자 (AWS, Google) 의 사고 통지
- AI provider (Google) 의 사용자 데이터 노출
- Audit log 위변조 가능성 발견
- TLS 인증서 만료 또는 유출

## 2. 단계 별 대응

### Phase 1 — 초기 대응 (0~1시간)

- [ ] **인지자 보고** → secops + DPO 즉시 통지 (Slack channel #incident + 전화)
- [ ] **사고 ID 부여** (`INC-YYYYMMDD-NN`)
- [ ] **현장 보존** — 로그 archive, 영향 시스템 격리
- [ ] **임시 차단** — 침해 추정 운영자 계정 정지, 의심 API key 회전
- [ ] **사고 대응팀 소집** — secops lead + DPO + 법무 + CEO

### Phase 2 — 영향 분석 (1~24시간)

- [ ] 침해된 개인정보 항목 식별 (data-inventory.md 매핑)
- [ ] 영향받은 사용자 수 산정
- [ ] 침해 시작 시각·종료 시각 (가능한 경우)
- [ ] 침해 경로·취약점 파악
- [ ] audit_logs hash chain 검증

### Phase 3 — 사용자/보호위원회 통지 (사고 인지 후 72시간 이내, 보호법 시행령 권고)

#### 3.1 사용자 통지 (모든 영향받는 사용자에게)

내용:
- 사고 일시
- 침해된 개인정보 항목
- 침해 경로 (구체적이지 않게)
- 회사의 대응 조치
- 사용자의 권리 안내 (피해 신고, 비밀번호 변경 등)
- 회사 연락처

수단:
- 이메일 (개별 발송)
- 앱 푸시 (긴급) — 영향받은 사용자만
- 공지사항 (앱 + 회사 홈페이지)

#### 3.2 보호위원회 신고

대상: **1천명 이상 영향받은 경우** (보호법 시행령 제39조 제2항)

- 보호위원회 신고 시스템 (개인정보분쟁조정위원회) 통해
- 사고 일시, 영향받은 정보 항목, 사용자 수, 대응 조치 보고

### Phase 4 — 사후 조치

- [ ] 침해 원인 제거 (취약점 패치, 정책 보강)
- [ ] 영향받은 운영자 계정 영구 회수 (해당 시)
- [ ] secret rotation (모든 운영자, 모든 API key)
- [ ] 사용자 비밀번호 강제 재설정 (해당 시)
- [ ] postmortem 작성 — `docs/incidents/INC-YYYYMMDD-NN-postmortem.md`
- [ ] 5W1H 분석 + 재발 방지 대책
- [ ] 사고 대응팀 디브리프
- [ ] 사용자 사후 안내 (해결 완료 시점)

### Phase 5 — 외부 감사

- [ ] 사고 처리 결과 보호위원회 보고
- [ ] 사용자 피해 보상 검토 (해당 시)
- [ ] 사내 보안 정책 갱신
- [ ] 분기 보안 review 안건 추가

## 3. 시나리오 별 대응 (예시)

### 시나리오 A: 운영자 계정 탈취 의심

1. 즉시 해당 운영자 모든 세션 revoke
2. 비밀번호·TOTP 재설정 요구
3. audit_logs 에서 해당 운영자의 최근 30일 활동 검토
4. 의심 mutation 있으면 rollback 검토

### 시나리오 B: SQL injection / API 취약점

1. 영향 API endpoint 비활성화
2. WAF rule 추가 (해당 시)
3. 패치 → 긴급 배포
4. DB query 로그 검토

### 시나리오 C: AWS 키 유출

1. 즉시 키 회전 (AWS IAM `aws iam update-access-key`)
2. CloudTrail 로그 검토 — 비정상 활동 확인
3. 영향받은 리소스 식별
4. 사용자 통지 (해당 시)

### 시나리오 D: Google Gemini 측 사고

1. Google 의 사고 통지 수신 후 24시간 이내 사용자 통지 검토
2. PII 마스킹 적용 여부 확인 (회사 측 redaction 효과)
3. AI 기능 일시 중단 (필요 시)
