## 1. ICONIA 인형 (IoT 디바이스) 안내

ICONIA 인형은 BLE (Bluetooth Low Energy) 및 Wi-Fi 무선 통신 기능을 탑재한 IoT 디바이스입니다.

| 항목 | 내용 |
|---|---|
| 무선 통신 | BLE 5.0 + Wi-Fi 2.4GHz |
| MCU | ESP32-S3 |
| 인증 | KC 적합성평가 (양산 출하 전 진행 — HD-03) |
| 안전 표준 | (HD-03 결정 후 명시) |

## 2. 페어링 및 프로비저닝 안내

ICONIA 앱과 인형을 연결하기 위해 다음 절차를 거칩니다.

1. **블루투스 권한 동의** — 앱이 인형을 BLE 로 검색·연결하기 위해 필요합니다.
2. **위치 권한 (Android 만)** — Android OS 의 BLE 스캔은 위치 권한을 요구합니다 (인형의 위치를 수집하지 않음).
3. **Wi-Fi 비밀번호 입력** — 인형이 가정 Wi-Fi 에 연결하기 위해 사용자의 Wi-Fi SSID 와 비밀번호를 BLE 를 통해 인형에 직접 전송합니다.

### Wi-Fi 비밀번호 저장 정책

- **앱**: 사용자가 입력한 Wi-Fi 비밀번호는 앱 내 SecureStore (iOS Keychain / Android EncryptedSharedPreferences) 에 암호화 저장됩니다 (사용자 디바이스 한정).
- **인형**: BLE 를 통해 수신한 Wi-Fi 비밀번호는 인형 내 보안 영역에 저장됩니다.
- **회사 서버**: Wi-Fi 비밀번호는 회사 서버에 전송되거나 저장되지 **않습니다** (코드 검증 후 확정 — HD-09).

## 3. 디바이스에서 수집하는 데이터

| 데이터 | 목적 | 보관 |
|---|---|---|
| 디바이스 식별자 (UUID, MAC) | 페어링·재연결 | 페어링 해제 시 익명화 |
| 펌웨어 버전 | OTA 호환성·롤백 | 영구 |
| 마지막 접속 시각 (last_seen_at) | 운영 모니터링 | 30일 |
| 배터리 잔량 (last_battery_percent) | 사용자 알림 | 30일 |
| BLE 진단 데이터 (RSSI, error code) | 운영 개선 (개인 비식별) | 30일 |
| 페어링/프로비저닝 상태 | 디버그 | 7일 |

## 4. 디바이스 안전 사용 안내

- 디바이스를 **분해·개조**하지 마세요. 보증 및 안전 인증이 무효화됩니다.
- 디바이스가 **과열·연기·이상 소음**을 보일 경우 즉시 전원을 차단하고 회사 고객지원에 문의하세요.
- 어린이의 손에 닿지 않는 곳에 보관하세요 (본 제품은 성인 대상 제품입니다).
- 디바이스는 가정 내 사용을 전제로 설계되었습니다.

## 5. 펌웨어 업데이트 (OTA) 안내

- 회사는 보안·기능 개선을 위해 정기적으로 펌웨어를 업데이트할 수 있습니다.
- 업데이트는 자동으로 진행되며, 사용자가 거부할 경우 일부 기능 사용이 제한될 수 있습니다.
- 업데이트는 **anti-rollback** 정책을 따르며, 이전 버전으로 임의 복구하지 않습니다 (보안 안전성 확보).
- 업데이트 실패 시 자동으로 이전 안정 버전으로 복귀합니다.

## 6. 디바이스 폐기 안내

이용자가 디바이스를 폐기할 경우:

1. 앱 설정 > 디바이스 페어링 해제
2. 인형 측 공장 초기화 (factory reset) — 사용자 매뉴얼 참조
3. 페어링 해제 후 회사 서버의 디바이스-사용자 매핑 정보는 익명화됩니다.
