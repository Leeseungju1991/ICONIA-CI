# 커머스 (display-only) 정책 (Draft — Legal Review Required)

> **본 문서는 초안이며 출시 전 법무 검토 후 최종본으로 게시됩니다.**

## 1. 정책 요지

ICONIA 앱은 상품·콘텐츠를 **안내** 만 제공하며, **결제·주문·장바구니·배송·환불 등의 거래 기능을 제공하지 않습니다**.

## 2. 사용자에게 표시되는 문구

앱 내 커머스 화면 진입 시 다음 안내가 표시됩니다.

> 본 화면은 상품 안내 목적이며, 앱 내에서 결제·주문은 제공하지 않습니다.
>
> 실제 구매는 회사 별도 채널을 통해 이루어집니다.

## 3. 허용 UI 요소

- 상품 사진·이름·간단 설명·가격(참고)
- 카테고리 분류
- "자세히 보기" / "제품 소개" 버튼
- "출시 예정" / "준비 중" 라벨

## 4. 금지 UI 요소

- "결제하기" / "주문하기" / "바로 구매"
- "장바구니" / "장바구니에 담기"
- "구매 확정" / "환불 신청"
- "배송 조회" / "주문 내역"
- 카드 결제 / PG 연동 / 정기결제

## 5. 데이터 처리

| 항목 | 위치 |
|---|---|
| 상품 데이터 (이름, 가격, 사진, 설명) | SERVER RDS `products` 테이블 — 운영자가 admin 콘솔에서 관리 |
| 사용자 행동 데이터 (조회) | 익명 통계로 활용 (PII 미포함) |
| 결제·청구 정보 | **수집하지 않음** |
| 배송지·연락처 | **수집하지 않음** |

## 6. 향후 결제 기능 도입 시

향후 앱 내 결제 기능을 도입하는 경우:

1. 별도 약관·환불·청약철회 정책 작성
2. 전자상거래법·통신판매업 신고 (HD-04)
3. PG 사 위탁 계약·국외이전 동의 (해당 시)
4. 결제 정보 PCI-DSS 수준 보호
5. 사용자 재동의 절차

이는 사업 정책 결정 사항이며 (HD-10), 결제 기능 도입은 본 문서 효력 범위를 벗어납니다.
