# 마케팅 정보 수신 동의 (Final Review-Ready)

> **본 문서는 법무 검토를 위한 final 수준 본문이다.**

## 1. 동의 항목 (선택)

| 항목 | 내용 | 비고 |
|---|---|---|
| 이메일 마케팅 수신 | 회사 서비스 안내·이벤트·할인 정보 | 선택 |
| 푸시 알림 마케팅 수신 | 동일 | OS 푸시 권한 + 본 동의 모두 필요 |
| 인앱 메시지 마케팅 | 동일 | 본 동의 시에만 표시 |
| (향후) SMS 마케팅 | 동일 | (현재 SMS 미사용 — 도입 시 별도 동의) |

## 2. 사용자에게 표시되는 동의 문구

> **(선택) 마케팅 정보 수신 동의**
>
> 회사가 운영하는 ICONIA 서비스의 새 기능·이벤트·할인·뉴스 등을 이메일·푸시 알림·인앱 메시지로 받기 동의합니다.
>
> - 동의하지 않으셔도 기본 서비스는 정상 이용 가능합니다.
> - 동의 후 언제든지 [설정 > 마케팅 수신] 또는 수신한 메시지의 [수신거부] 링크로 동의를 철회할 수 있습니다.

## 3. 동의 시점

- **회원 가입 시**: 선택 동의 화면에서
- **설정 화면에서**: 언제든지 동의·철회

## 4. 동의 이력 보관

- 동의/철회 시각, 동의 항목, IP, User-Agent 를 `consent_records` 테이블에 영구 기록 (5년)
- 정보통신망법 제50조 (영리 목적의 광고성 정보 전송 제한) 의 동의 입증 자료

## 5. 영리 목적 광고성 정보 전송 규정 (정보통신망법 §50)

회사는 다음을 준수합니다.

| 항목 | 회사 조치 |
|---|---|
| 광고성 정보 전송 시 사전 동의 | 본 정책 §1 의 동의 후만 전송 |
| 광고임을 명시 | 메시지 제목/본문에 "(광고)" 명시 |
| 무료 수신거부 수단 제공 | 모든 메시지에 [수신거부] 링크 |
| 야간 시간 전송 제한 | 21:00~08:00 마케팅 푸시 전송 안 함 (긴급 공지 제외) |
| 동의 철회 시 즉시 중단 | 철회 즉시 발송 큐에서 제외 |

## 6. 마케팅 푸시 발송 운영 (Admin Console)

- 운영자는 ADMIN `dashboard/notice-push` 에서 푸시 캠페인 작성
- audience 옵션:
  - `all` — 마케팅 동의자만 자동 필터
  - `staff` — 회사 직원 (운영자 테스트)
  - `beta` — 베타 테스트 사용자
- 발송 전 운영자 confirm 필수
- 발송 결과 audit log 기록 (5년)

## 7. 거부 사용자 제외 정책

- `users.marketing_opt_in = false` 인 사용자는 마케팅 푸시 audience 에서 자동 제외
- audit log 검증: 발송 시점 `marketing_opt_in` 확인

## 8. 향후 마케팅 동의 항목 추가 시

신규 마케팅 항목 (예: SMS, 외부 채널 광고 등) 도입 시 다음 절차를 따릅니다.

1. 사용자 재동의 요청 (ReconsentModal)
2. 동의 이력 신규 row 자동 추가
3. 거부 사용자는 신규 항목 비활성
