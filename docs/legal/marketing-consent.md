# 마케팅 정보 수신 동의

## 1. 동의 항목 (선택)

| 항목 | 내용 | 비고 |
|---|---|---|
| 이메일 마케팅 수신 | 회사 서비스 안내·이벤트·할인 정보 | 선택 |
| 푸시 알림 마케팅 수신 | 동일 | OS 푸시 권한 + 본 동의 모두 필요 |
| 인앱 메시지 마케팅 | 동일 | 본 동의 시에만 표시 |
| SMS 마케팅 | 동일 | 현재는 제공하지 않습니다. SMS 마케팅을 시작할 때 별도로 동의를 받습니다. |

## 2. 사용자에게 표시되는 동의 문구

> **(선택) 마케팅 정보 수신 동의**
>
> 회사가 운영하는 ICONIA 서비스의 새 기능·이벤트·할인·뉴스 등을 이메일·푸시 알림·인앱 메시지로 받기 동의합니다.
>
> - 동의하지 않으셔도 기본 서비스는 정상 이용 가능합니다.
> - 동의 후 언제든지 [설정 > 마케팅 수신] 또는 수신한 메시지의 [수신거부] 링크로 동의를 철회할 수 있습니다.

## 3. 동의 시점

- **회원 가입 시**: 선택 동의 화면에서
- **설정 화면에서**: 언제든지 동의·철회

## 4. 동의 이력 보관

- 동의/철회 시각, 동의 항목, 접속 정보를 5년간 기록 보관
- 정보통신망법 제50조 (영리 목적의 광고성 정보 전송 제한) 의 동의 입증 자료

## 5. 영리 목적 광고성 정보 전송 규정 (정보통신망법 §50)

회사는 다음을 준수합니다.

| 항목 | 회사 조치 |
|---|---|
| 광고성 정보 전송 시 사전 동의 | 본 정책 §1 의 동의 후만 전송 |
| 광고임을 명시 | 메시지 제목/본문에 "(광고)" 명시 |
| 무료 수신거부 수단 제공 | 모든 메시지에 [수신거부] 링크 |
| 야간 시간 전송 제한 | 21:00~08:00 마케팅 푸시 전송 안 함 (긴급 공지 제외) |
| 동의 철회 시 즉시 중단 | 철회 즉시 발송 큐에서 제외 |

## 6. 마케팅 푸시 발송 운영 (관리자 콘솔)

- 운영자는 관리자 콘솔의 공지/푸시 화면에서 푸시 캠페인을 작성합니다.
- 발송 대상 옵션:
  - 전체 사용자 — 마케팅 수신 동의자만 자동 필터
  - 회사 직원 — 운영자 테스트
  - 베타 테스트 사용자
- 발송 전 운영자 확인 필수
- 발송 결과 감사 로그 기록 (5년)

## 7. 거부 사용자 제외 정책

- 마케팅 수신에 동의하지 않은 사용자는 마케팅 푸시 발송 대상에서 자동 제외됩니다.
- 발송 시점의 동의 여부를 확인하여 처리합니다.

## 8. 마케팅 동의 항목 추가 시

새로운 마케팅 항목 (예: SMS, 외부 채널 광고 등) 을 도입할 때에는 다음 절차를 따릅니다.

1. 사용자 재동의 요청
2. 동의 이력 신규 추가
3. 거부 사용자는 신규 항목 비활성
