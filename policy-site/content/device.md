# 디바이스 연결·데이터 안내 (Final Review-Ready)

> **본 문서는 법무·인증 전문가 검토를 위한 final 수준 본문이다.** HW 정본 (`1. HW/docs/legal/data-collection.md`) + companyInfo source 통합.

## 1. ICONIA 인형 안내

ICONIA 인형은 BLE 5.0 + Wi-Fi 2.4GHz 무선 통신 기능을 탑재한 IoT 디바이스입니다.

| 항목 | 값 |
|---|---|
| 무선 통신 | BLE 5.0 + Wi-Fi 2.4GHz |
| MCU | ESP32-S3 |
| 카메라 | 사용자 손길 터치 시 1장 캡처 (RAM only, 업로드 후 즉시 폐기) |
| 마이크 | 미탑재 |
| GPS | 미탑재 |
| 인증 | KC 적합성평가 (양산 출하 전 진행) |

## 2. 페어링 및 프로비저닝

ICONIA 앱과 인형을 연결하기 위해 다음 절차를 거칩니다.

1. **블루투스 권한 동의** — 앱이 인형을 BLE 로 검색·연결하기 위해 필요합니다.
2. **위치 권한 (Android 만)** — Android OS 의 BLE 스캔은 위치 권한을 요구합니다. **인형의 위치를 수집하지 않으며**, 위치 권한은 BLE 스캔 OS 정책상의 형식적 요건입니다.
3. **Wi-Fi 비밀번호 입력** — 인형이 가정 Wi-Fi 에 연결하기 위해 사용자의 Wi-Fi SSID 와 비밀번호를 BLE 를 통해 인형에 직접 전송합니다.

### Wi-Fi 비밀번호 저장 정책 (확정)

- **앱**: 사용자가 입력한 Wi-Fi 비밀번호는 앱 내 SecureStore (iOS Keychain / Android EncryptedSharedPreferences) 에 암호화 저장됩니다 (사용자 디바이스 한정).
- **인형**: BLE 를 통해 수신한 Wi-Fi 비밀번호는 인형 내 NVS (AES-256-GCM 암호화) 에 저장됩니다. 정본: `1. HW/docs/legal/data-collection.md` §1.
- **회사 서버**: Wi-Fi 비밀번호는 **회사 서버에 전송되거나 저장되지 않습니다**. 펌웨어 정본 `iconia_app.cpp` 의 코드 검증 결과 — 송신 경로 없음.

## 3. 디바이스에서 수집·송신하는 데이터 (HW 정본 §1)

| # | 데이터 | 디바이스 측 보관 | 송신 |
|---|---|---|---|
| 1 | 카메라 JPEG (손길 터치 시 1장) | RAM only — 업로드 직후 즉시 폐기 | TLS 1.2 + ACM 인증서 + (선택) SPKI 핀 → 회사 운영 서버 → AWS S3 (서울) |
| 2 | 터치 이벤트 (좌/우/타이밍) | RTC slow-memory (중복 방지 키) | 이벤트 메타와 함께 송신 |
| 3 | 배터리 잔량·상태 (6단계) | NVS 마지막 값 | 이벤트 메타와 함께 송신 |
| 4 | Wi-Fi SSID/비밀번호 | NVS (AES-256-GCM 암호화) | **송신 없음** |
| 5 | 디바이스 식별자 (device_id_hash, 8 hex) | NVS / RTC — 부팅 시 eFuse MAC → HKDF 유도 | 페어링·이벤트 시 |
| 6 | panic_log / boot invariant mask | NVS 1개 키 | 부팅 직후 |
| 7 | OTA telemetry (단계·RSSI·진척) | RTC slow-memory 큐 | OTA 진행 중 |
| 8 | 에러 보고 | RAM throttle 카운터 | 5/5min throttle |
| 9 | 펌웨어 버전·HW Rev·모델명 | 펌웨어 매크로 정본 | 매 요청 헤더 |

## 5. 디바이스 안전 사용 안내

- 디바이스를 **분해·개조**하지 마세요. 보증 및 안전 인증이 무효화됩니다.
- 디바이스가 **과열·연기·이상 소음**을 보일 경우 즉시 전원을 차단하고 회사 고객지원(support@dollsoom.com)에 문의하세요.
- 어린이의 손에 닿지 않는 곳에 보관하세요 (본 제품은 성인 대상 제품입니다).
- 디바이스는 가정 내 사용을 전제로 설계되었습니다. 야외·자동차 내부·고온 환경(60℃ 이상) 등에서 사용하지 마세요.
- 배터리 충전 시 정품 충전기·케이블만 사용하세요.

## 6. 펌웨어 업데이트 (OTA) 안내

- 회사는 보안·기능 개선을 위해 정기적으로 펌웨어를 업데이트할 수 있습니다.
- 업데이트는 자동으로 진행되며, 사용자가 거부할 경우 일부 기능 사용이 제한될 수 있습니다.
- 업데이트는 **anti-rollback** 정책을 따르며, 이전 버전으로 임의 복구하지 않습니다.
- 업데이트 실패 시 자동으로 이전 안정 버전으로 복귀하며, 사용자 데이터는 영향받지 않습니다.

## 7. 디바이스 폐기·소유권 이전 (AS 핸드오프, HW 정본 §5)

이용자가 디바이스를 폐기하거나 다른 사용자에게 양도할 경우, 다음 절차로 모든 사용자 데이터가 안전하게 zeroize 됩니다:

1. NVS `wifi_ssid` / `wifi_pw` zeroize
2. BLE bond 정보 zeroize
3. `device_id_hash` 재유도 시드 회전
4. panic_log / OTA 결과 삭제
5. session counter 초기화
6. 회사 AS endpoint 에 zeroize 완료 보고
7. BLE 광고명 `ICONIA-AS-XXXX` 으로 분기 후 새 owner 페어링 대기

factory seed / eFuse SECURE_VERSION 은 zeroize 대상이 아닙니다 (HW 정체성 보존).
