# 커머스 정책

## 1. 정책 요지

ICONIA 앱은 상품·콘텐츠를 **안내** 만 제공하며, 결제·주문·장바구니·배송·환불 등의 거래 기능을 제공하지 않습니다.

## 2. 사용자에게 표시되는 안내 문구

앱 내 [상품 안내] 화면 진입 시 다음 안내가 표시됩니다.

> 본 화면은 상품 안내 목적이며, 앱 내에서 결제·주문은 제공하지 않습니다.
>
> 실제 구매는 회사가 별도로 운영하는 외부 채널을 통해 이루어집니다. (회사 홈페이지 또는 안내된 외부 구매처)

## 3. 허용 UI 요소

- 상품 사진·이름·간단 설명·가격(참고)
- 카테고리 분류
- 검색·필터
- 즐겨찾기 (개인 식별 정보와 분리)
- "자세히 보기" / "제품 소개" 버튼
- "출시 예정" / "준비 중" 라벨
- (선택) 외부 구매처 안내 (회사 홈페이지 등)

## 4. 금지 UI 요소

| 금지 문구 | 허용 대체 |
|---|---|
| 결제하기 | 자세히 보기 |
| 주문하기 | 제품 소개 |
| 바로 구매 | 외부 구매처 안내 |
| 장바구니 / 장바구니에 담기 | (사용하지 않음) |
| 구매 확정 / 환불 신청 | (사용하지 않음) |
| 배송 조회 / 주문 내역 | (사용하지 않음) |

## 5. 데이터 처리

| 항목 | 처리 |
|---|---|
| 상품 데이터 (이름·가격·사진·설명·카테고리·재고) | 운영자가 관리자 페이지에서 관리 |
| 사용자 조회 통계 | 익명 통계 (개인 식별 정보 미포함) |
| 결제·청구 정보 | **수집하지 않음** |
| 배송지·연락처 | **수집하지 않음** |
| 즐겨찾기 | 사용자 식별 정보와 연결되어 저장 — 탈퇴 시 익명화 |

## 6. 향후 결제 기능 도입 시

향후 앱 내 결제 기능을 도입하는 경우 다음을 준비합니다.

1. 별도 약관·환불·청약철회 정책 작성
2. 「전자상거래 등에서의 소비자보호에 관한 법률」 준수 + 통신판매업 신고
3. PG 사 위탁 계약·국외이전 동의 (해당 시)
4. 결제 정보 PCI-DSS 수준 보호
5. 사용자 재동의 절차
6. ADMIN 콘솔의 주문·결제·환불 관리 페이지 추가
7. 관련 법령에 따른 추가 정책 정비

이는 사업 정책 결정 사항이며, 결제 기능 도입은 본 정책의 효력 범위를 벗어납니다.
